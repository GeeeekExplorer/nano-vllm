# AVG 本地模型生成系统交付总览

## 项目目标

本项目目标是为 AVG 游戏建立一套可本地部署、可扩展、可解释的文本生成链路。系统将玩家行为、剧情节点和外部星体影响整合为模型可理解的约束，再由本地大语言模型生成旁白和角色台词。

## 一页式架构

```text
玩家操作记录
  -> 玩家行为量化
  -> AVG 状态参数
  -> 接收外部星体影响 AstroInfluence
  -> 剧情节点收敛 PlotPoint
  -> Prompt 约束生成
  -> nano-vLLM 本地模型推理
  -> AVG 旁白和台词
```

## 模块边界

`nano-vLLM` 负责模型加载、推理加速、请求调度和文本生成。AVG 模块负责把玩家行为、剧情节点和星体影响转换为 prompt。星体运动拟合、游戏内星体运作和随机化算法由外部模块独立实现，可以输出标准 `AstroInfluence`，也可以先输出普通字典 payload。

这种边界让三类工作可以并行推进：模型推理可以独立优化，剧情规则可以独立调整，星体随机化模块也可以独立迭代。未知字段会先保存在 metadata 中，后续联调稳定后再提升为正式字段。

## 当前风险与后续计划

| 项目 | 当前情况 | 后续建议 |
|---|---|---|
| 生成质量 | 依赖模型能力、prompt 设计和剧情节点配置 | 用真实剧情素材做人工评估和 prompt 调优 |
| 硬约束 | v1 通过 prompt 表达禁止词和必需词 | 后续在 sampler 或 decode 后处理中实现强制约束 |
| 星体算法 | 当前只定义接入接口 | 等外部星体模块完成后做联调 |
| 外部数据类型 | 已支持字典输入和未知字段透传 | 联调后沉淀正式字段清单 |
| 复现能力 | 已支持 seed 设置 PyTorch 随机种子 | 在完整推理环境中验证 GPU 采样复现效果 |
| 游戏接入 | 已提供 `generate_with_plan()` 调试结果 | 后续对接存档、剧情编辑器或运行时日志 |

## 可对外汇报文本

当前版本已经完成本地模型推理底座和 AVG 文本生成适配。系统可以将玩家行为量化为剧情状态，接收外部星体运动随机化模块输出的影响向量，并根据状态收敛到对应剧情节点，最终调用本地模型生成旁白和角色台词。

该方案采用清晰的分层架构：底层 `nano-vLLM` 负责高效推理，上层 AVG 模块负责剧情约束，星体运动和随机化算法作为外部模块接入。这样的设计便于后续扩展、调试和替换模型，也能降低游戏逻辑与模型推理之间的耦合度。
